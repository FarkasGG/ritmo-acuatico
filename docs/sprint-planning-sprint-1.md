\# Sprint Planning – Sprint 1

\*\*Proyecto:\*\* Ritmo Acuático  

\*\*Metodología:\*\* Scrum  

\*\*Tecnología:\*\* React (Vite)



\## 1. Objetivo del Sprint

Desarrollar una primera versión funcional del sitio web Ritmo Acuático utilizando React, incluyendo una landing page, identidad visual básica, catálogo de productos y diseño responsive para distintos dispositivos.



\## 2. Duración y capacidad del Sprint

\- Duración del Sprint: 2 semanas

\- Integrantes: 1

\- Capacidad estimada: 30–32 horas



\## 3. Product Backlog (historias relevantes)

\- PB-01 Landing page del sitio

\- PB-02 Catálogo de productos

\- PB-03 Diseño responsive

\- PB-06 Identidad visual del sitio



\## 4. Sprint Backlog (Sprint 1)

\- PB-01 Landing page del sitio

\- PB-06 Identidad visual del sitio

\- PB-02 Catálogo de productos

\- PB-03 Diseño responsive



\## 5. Definition of Done (DoD)

Una historia se considera terminada cuando:

\- El proyecto compila y se ejecuta correctamente con `npm run dev`.

\- La landing page muestra información clara de la marca.

\- El catálogo lista productos con imagen, nombre y precio.

\- El diseño se adapta correctamente a móvil, tablet y escritorio.

\- No existen errores críticos en consola.



\## 6. Desglose de tareas por historia



\### PB-01 Landing page

\- Estructura JSX de la página principal

\- Contenido textual y secciones

\- Botón de llamada a la acción



\### PB-06 Identidad visual

\- Definición de paleta de color

\- Tipografía base

\- Estilos generales del sitio



\### PB-02 Catálogo de productos

\- Estructura de datos de productos

\- Componente ProductCard

\- Renderizado dinámico del catálogo

\- Integración de imágenes reales desde src/assets



\### PB-03 Diseño responsive

\- Media queries para tablet

\- Media queries para móvil

\- Pruebas en distintos tamaños de pantalla



\## 7. Criterios de aceptación

\- La landing page es visible y clara.

\- El catálogo muestra imágenes reales, nombre y precio.

\- El sitio es usable en móvil, tablet y escritorio.

\- La identidad visual es consistente.



\## 8. Sprint Review (resultado)

Se entregó una versión funcional del sitio web Ritmo Acuático con landing, identidad visual, catálogo y diseño responsive, cumpliendo los objetivos del Sprint 1.



\## 9. Sprint Retrospective (reflexión)

del mismo. Esto me ayudara a mejorar la estimacion del tiempo en futuros srpints
